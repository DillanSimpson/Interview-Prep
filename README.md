# MCQ Practice — LeetCode‑style Java Problems

This project contains a curated set of **classic interview problems** that frequently appear in large fintech/back-end interviews.  
Each problem has:
- A **TODO** description explaining constraints and hints.
- A **method stub** you must implement.
- **Unit tests** that capture the expected behavior and edge cases.

> These are common patterns seen across many companies. They are not proprietary. Use them to sharpen fundamentals.

## Problems
1. Two Sum (hash map)
2. Longest Substring Without Repeating Characters (sliding window)
3. Valid Parentheses (stack)
4. Merge Intervals (sorting + sweep)
5. LRU Cache (O(1) get/put)
6. Top K Frequent Elements (heap / bucket sort)
7. Kth Largest in a Stream (min-heap)
8. Meeting Rooms II (min-heap of end times)
9. Product of Array Except Self (prefix/suffix, no division)
10. Rotting Oranges (multi-source BFS)

## Run
```bash
mvn -q test
```

Start by opening the `TODO` blocks in `src/main/java/.../problems/*` and implement methods until all tests go green.
